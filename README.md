# Основная часть

## 1. `IndexError: pop from empty list` при пустом выражении
- Причина: в конце `calculate` выполнялся `stack.pop()`, даже когда стек пуст
- Пример: `input_string=''`
- Исправление: добавлена проверка:
```python
if len(stack) == 0:  
        raise CalcException("Пустое выражение")
```
Вывод консоли в файле `dump1.txt`.

## 2. `IndexError: string index out of range` для любого односимвольного оператора

Причина: использовалось `match s_tmp[1]`, но у строки длины 1 не существует индекс 1.
Пример: `input_string='0^0'`

Исправление: `match s_tmp[1]:` заменено на `match s_tmp:`.

Вывод консоли в файле `dump2.txt`.

## 3. `NotImplementedError` для оператора `!`
Причина: оператор объявлен как допустимый, но внутри выбрасывался `NotImplementedError`.
Пример: `input_string='0!0'`

Исправление: добавлена реализация унарного оператора '!', из бинарных операторов '!' был убран
```python
if len(s_tmp) == 1 and is_op(s_tmp):
    if s_tmp == '!':
        if len(stack) < 1:
            raise CalcException("Неверное количество данных в стеке для операции \"!\"")

        a = stack.pop()

        if a < 0 or not float(a).is_integer():
            raise CalcException("Факториал определен только для неотрицательных целых чисел")

        n = int(a)
        if n > 170:
            raise OverflowError("Слишком большое значение для факториала")

        res = 1.0
        for i in range(2, n + 1):
            res *= i

        stack.append(res)
        continue
```

Вывод консоли в файле `dump3.txt`.

## 4. `ZeroDivisionError` при возведении нуля в отрицательную степень
Причина: выражение вида 0 ** -1 в Python вызывает `ZeroDivisionError`.
Пример: `input_string='0^(0-1)'`

Исправление: обернуто в try/except:
```python
try:
    a **= b
except ZeroDivisionError:
    raise CalcException("Нельзя возводить ноль в отрицательную степень")
```
Вывод консоли в файле `dump4.txt`.
